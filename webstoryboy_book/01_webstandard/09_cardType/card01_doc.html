소개

안녕하세요!. 웹스토리보이입니다. 이번 주제는 카드 유형입니다. 카드 유형은 사이트 제작에 있어 필수적으로 들어가는 레이아웃입니다. 3단, 4단 유형을 많이 사용하며, 콘텐츠를 효과적으로 보여주는 구조입니다.

이론

이번 카드 유형에서는 만들어 볼 디자인은 이것입니다. 어렵지 않으니 하나씩 따라하시면 만들 수 있습니다. 그럼 시작해볼까요? 우선 card01.html 페이지를 만들고 !를 치고 tab을 누르겠습니다. 언어를 ko로 변경하고 title은 카드 유형01로 변경하겠습니다. 

<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>카드 유형01</title>
    <style>
        * {
            margin: 0;
            padding: 0;
            box-sizing: border-box;
        }
        .container {
            width: 1160px;
            margin: 0 auto;
            padding: 0 20px;
            background: rgba(0,0,0,0.3);
        }
        .card-wrap {
            padding: 120px 0;
            background: rgba(0,0,0,0.1);
        }
    </style>
</head>
<body>
    <section class="card-wrap">
        <div class="container">

        </div>
    </section>
</body>
</html>

문서 세팅1을 완료하고, section 태그를 이용하여, 콘텐츠를 설정합니다. 시멘틱 태그2를 이용하여 카드 영역임을 표현합니다. 클래스 이름은 card-wrap이라고 설정하겠습니다.

가운데 영역을 설정하기 위해서 container를 클래스를 만들고, width:1160px을 설정합니다. 블록구조를 가운데 오기 위해서 margin: 0 auto를 설정하고, 영역 확인을 위해 백그라운드를 넣어줍니다.

head 태그 안에 style 스타일을 선언하고, CSS 초기화3 작업을 하겠습니다. 전체(*)선택자를 이용하여 마진, 패딩, 박스 사이즈를 초기화하겠습니다.

card__inner 박스를 만들고, 자식으로 3개의 박스를 만들었습니다. 이미지, 제목, 설명, 버튼을 만들어 추가하였습니다. 

.container {
    width: 1160px;
    margin: 0 auto;
    padding: 0 20px;
    background: rgba(0,0,0,0.1);
}
.card-wrap {
    padding: 120px 0;
    text-align: center;
    background: rgba(0,0,0,0.1);
    font-family: "GmarketSans";
}
.card-wrap h2 {
    font-size: 50px;
    margin-bottom: 20px;
    font-weight: 400;
}
.card-wrap p {
    font-size: 22px;
    line-height: 1.5;
    font-weight: 300;
    color: #777;
    margin-bottom: 80px;
}
.card__inner {
    display: flex;
    justify-content: space-between;
    text-align: left;
}
.card__inner > div {
    width: 32%;
    background: #fff;
}
.card__inner > div img {
    width: 100%;
}

.card-wrap p에 margin-bottom: 80px;를 추가합니다. .card__inner 자식으로 3개의 div가 있습니다. .card__inner > div 바로 밑에 있는 자식만 표현하기 위해 > 기호를 사용하였습니다.

display: flex;를 사용하여, 자식 박스를 가로로 정렬하고, width: 32%; 값을 설정하여 크기를 맞추었습니다. img 요소에는 width: 100%;을 주어 영역 크기에 맞추었습니다.

똑같나요? 하지만 여기서 문제가 하나 생겼습니다. 눈치 채셨나요?

처음에 코딩할 때 .card-wrap p라고 코딩을 하였습니다. card-wrap 안에 있는 모든 p 태그를 선택하기 때문에 card__inner 안에 있는 p 태그 한테도 영향을 줍니다. 그래서 결과를 확인해보면 설명 밑에 간격이 많이 있습니다. 이걸 수정하려면 .card-wrap p 요소를 .card-wrap > p 이렇게 변경하면, 해결 할 수 있습니다. 하지만 저는 이름을 체계화하여 영향을 주지 않고, 재활용 할 수 있도록 section 클래스를 추가하여 작업하겠습니다. 섹션마다 보여주는 제목과 설명은 모든 섹션이 동일하기 때문에 이름을 체계회하겠습니다.

별반 차이 없는 효과같지만, 이런식으로 이름을 효율적으로 활용하고, 효율성 있게 작업하는게 중요합니다. 지금은 이런 작업을 왜 할까? 생각할 수 있지만, 나중에 사이트 작업할 때 왜 이렇게 했는지 알 수 있습니다. 끝까지 잘 따라하신다면 저의 깊은 생각을 알 수도 있을겁니다. 🥴 폰트 역시 따로 클래스를 만들어서 작업하였습니다. 혹시 나중에 이 폰트 말고 다른 폰트 쓰고 싶다면 클래스 이름만 변경하면, 수정이 용이하겠죠!

클래스 이름도 변경하였으니, CSS도 달라지겠죠! 최종 결과 및 소스를 확인해보시고 수정해주세요!

어떤가요? 조금 달라진거 같나요? 저도 예전에는 왼쪽처럼 코딩을 했습니다. 왼쪽 코딩이 틀린건 아닙니다. 훨씬 더 심플하고 간단하게 표현 할 수 있죠? 하지만 웹 표준을 준수하고, 접근성을 향상시키기 위해서는 오른쪽 방법이 올바른 방법입니다.

card 영역은 시멘틱 태그 <article>로 대처 하였고, 세부적으로 card__header와 card__body로 구분하고, 안에 제목과 설명, 이미지 등으로 구성하였습니다. 버튼 말고, 날짜나 시간, 키워드 같은 것을 추가한다면 card__footer를 만들 수도 있습니다. 이미지를 인용하기 때문에 <figure> 태그도 사용하였습니다.

CSS도 변경하였습니다. div, p 기본 태그를 사용하는건 충돌이 날 가능성이 많습니다. 큰 프로젝트 작업을 하거나 여러명에서 코딩을 한다면, 충돌날 가능성이 높기 때문에 영역의 이름을 정확하게 적어주는 것도, 가독성을 높이고, 유지 보수에 더 도움이 될 수 있기 때문에 오른쪽처럼 작업해주는게 더 좋습니다.

그럼 이제 세부적인 작업을 해보겠습니다.

설명 부분에 글씨 줄을 통일 시켜주겠습니다. 글을 한 줄만 나오게 하거나, 여러 줄을 나오게 설정할 수 있습니다. 자세한 건 한줄 효과5를 확인해보세요! 저희는 다섯줄 효과를 적용할 것입니다. 

overflow: hidden;
text-overflow: ellipsis;
display: -webkit-box;
-webkit-line-clamp: 5; 
-webkit-box-orient: vertical;

overflow: hidden;는 내가 지정한 영역 이외에는 안보여주는 역할을 합니다. text-overflow: ellipsis는 끝 부분을 ...으로 변경해줍니다. display: -webkit-box;은 박스 구조로 성질을 변경해 줍니다. -webkit-이라는 벤더프리픽스6는 아직 베타 버전이기 때문에 임시로 사용하는 용도입니다. 시간이 흐르고 베타 버전이 끝나면 안 붙여도 됩니다. -webkit-line-clamp: 5;는 줄 수를 설정하고, -webkit-box-orient: vertical;는 방향을 설정합니다. 더 궁금한 부분이 있다면 한줄 효과와 벤더프리픽스를 참고하세요!
