이론

우리가 이번에 만들어 볼 유형은 이런 디자인입니다. 배경에 이미지가 있고, 가운데 영역에 텍스트가 있는 구조입니다. 지금까지 잘 따라 오셨다면 폴더 구조는 이렇게 되어 있을겁니다. webstandard 폴더에 06_banner01.html 페이지를 만들겠습니다.

시멘틱 태그1 <section> 이용하여 배너 영역임을 표현합니다. 클래스 이름은 banner-wrap이라고 설정하겠습니다.

banner-wrap 클래스 이미지를 넣겠습니다. 웹 사이트를 만들 때 이미지를 표현하는 방법2은 3가지 있습니다. 첫번째는 의미가 없는 이미지는 css를 통해 표현하고, 두번째 의미가 있는 이미지는 img 태그를 통해 표현합니다. 이게 이미지를 표현 하는 가장 기본적인 방법이고, 세번째 방법은 웹 표준을 준수하기 위한 방법으로 추후에 배우게 될 것입니다. 우선 그냥 해보겠습니다. 해보면 알게됩니다. ☺️ 

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>배너 유형01</title>
    <style>
        .banner-wrap {
            background-image: url(../../assets/img/banner_bg01@2x.jpg);
            background-position: center center;
            background-size: cover;
            background-repeat: no-repeat;
            height: 500px;
        }
    </style>
</head>

디자인 작업을 해주는 CSS는 선언하는 규칙3이 3가지 정도 있습니다. 우리는 첫 번째 방법으로 head 요소 안에 style을 선언하고 작업하겠습니다.

background-image 속성을 통해 이미지를 넣으면 됩니다. 우선 영역이 없기 때문에 가상의 영역을 잡아주기 위해서 height: 500px을 추가할 것입니다. 이미지 파일을 넣을 때는 상대주소4를 이용하여 넣을 것입니다. background-position 속성을 통해 이미지의 위치를 설정하고, background-size를 통해 이미지를 영역 크기에 맞추고, background-repeat를 통해 이미지를 한번만 나오게 할 것입니다. 이렇게 백그라운드 속성5은 다양하게 있으며, 상황에 따라 적절하게 사용하면 됩니다. 아직 처음이라 이런 속성들을 전부 알 수 없습니다. 하나씩 따라하면 점점 익숙해지고, 알게 될 것입니다.

여기까지 잘 따라하셨다면 결과를 확인해야 합니다. 결과는 VScode의 Live Server6를 통해 확인하셔도 되고, 우리가 만든 파일을 클릭해서 확인을 해도 됩니다.

결과를 확인해보면 여백이 생기는 것을 확인 할 수 있습니다. 초기에 사이트 작업시 모든 태그에는 보이지 않는 여백이 존재합니다. 이런 부분을 초기화7 시켜주어야 합니다. 

* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

지금 간단하게 바깥쪽 여백, 안쪽 여백, 패딩 값 포함 유무 정도만 설정하였습니다. 실전 사이트에서 더 디테일하게 설정할 것입니다. 이렇게 하면 여백없이 이미지가 잘 나올 것입니다. 제가 작업한거랑 한번 확인해보세요! 🥺

그럼 이번에는 banner-wrap 요소를 작업해 보겠습니다. 

<div class="banner__inner">
    <h3>유튜버 웹보이</h3>
    <p>

제목은 h3 태그, 설명은 p 태그, 배너 영역 글씨는 마땅한게 없으니 span 태그를 사용하겠습니다. span 태그는 마땅한게 없을 때 쓰는 인라인 요소입니다. 물론 자기가 생각하기에 또는 구조상 의미상 설명 부분이 맞다고 생각하면 p 태그를 사용하셔도 됩니다. 항상 얘기 했듯이 정답은 없습니다. 가장 효과적인 방법과 효율적인 방법으로 작업하시면 됩니다. 가장 효과적인 방법과 효율적인 방법은 경험을 통해서 조금씩 생기니, 천천히, 조금씩, 반복적으로 따라하시면 됩니다. 🥶

이번에는 웹 폰트8를 설정하겠습니다. 

<link href="https://webfontworld.github.io/gmarket/GmarketSans.css" rel="stylesheet">

웹 폰트를 설정하는 방법도 여러가가 있지만, 우리는 링크 방식을 이용해서 title 태그 밑에 추가하겠습니다. 웹 폰트를 사용하는 이유는 사용자 컴퓨터에 해당 폰트가 없으면, 해당 폰트를 다운하고 보여주는 방식입니다. 사이트 제작시 폰트는 아주 중요합니다. 폰트가 깨진다면 디자인 많이 이상해 질 것입니다. 그래서 저는 웹 폰트를 많이 사용합니다. 하지만 파일 용량이 크기 때문에 처음에 시작할 때는 시간이 지연 될 수 있는 단점이 있습니다. 그렇지만 자주 사용하는 폰트나 용량을 최적화 시킨 폰트를 사용한다면 크게 문제되지 않습니다.

이번에는 이름을 부여하겠습니다. 

<div class="banner__inner">
    <h3 class="title">유튜버 웹보이</h3>
    <p class="desc">
        더 다양한 강의는 유튜브를 통해 제공하고 있습니다.
        <a href="#">youtube.com/c/Webstoryboy</a>
    </p>
    <span class="sub">배너 영역01</span>
</div>

클래스 이름을 주는 것은 아주 중요합니다. 사이트가 복잡해지나, 여러 명에서 작업시 클래스 이름이나 아이디는 신중하게 정해야 합니다. 여러분들의 스킬이 쌓이면 이해가 될 것입니다. 그럼 이름을 지웠으니, 선택해서 속성을 주겠습니다. 이것을 선택자9라고 하죠! 

.banner__inner {
    padding: 120px 0;
    text-align: center;
    color: #fff;
    font-family: "GmarketSans";
}

위 아래로 여백을 주고, 글씨는 가운데 정렬하겠습니다. 모든 글씨는 하얀색으로 설정하고, 폰트는 지마켓으로 하겠습니다. 

.banner__inner .title {
    font-size: 50px;
    font-weight: 300;
    margin-bottom: 40px;
}
.banner__inner .desc {
    font-size: 24px;
    line-height: 1.5;
    font-weight: 300;
    margin-bottom: 70px;
}
.banner__inner .desc a {
    display: block;
    font-size: 24px;
    line-height: 1.5;
    font-weight: 300;
    color: #fff;
    text-decoration: none;
    margin-bottom: 70px;
}
.banner__inner .sub {
    font-size: 16px;
    font-weight: 300;
    text-decoration: underline;
    text-underline-position: under;
}

각각의 속성에 폰트 사이즈, 줄 간격, 폰트 두께, 바깥쪽 여백, 밑줄 속성을 주겠습니다. 각각의 사이즈는 피그마 또는 제플린을 확인하시면 됩니다. .banner__inner .desc a 속성은 블록구조로 변경하시면, 줄바꿈 되는 것을 확인 할 수 있습니다. .banner__inner에 color 속성을 적용하였지만, a 태그에만 적용되지 않습니다. a 요소에는 따로 설정해주셔야 합니다. 또한 기본 값이 파란색 글씨와 밑줄이 생기니 이 부분도 수정했습니다.

여기까지 하시면 1차적으로 완성한 것입니다. 이번에는 조금더 각색을 해보겠습니다. 재활용 가능하도록 클래스 이름을 변경하고, 공통요소들은 따로 설정하겠습니다. 왜 이렇게 하는지는 당장에 바로 설명하기가 힘듭니다. 사이트를 다 완성하면 왜 이렇게 했는지 이해가 될 것입니다.

우선 section 태그를 사용하게 되면, 제목을 나타내는 <h1> ~ <h6> 태그가 포함되어야 합니다. 

<section class="banner-wrap">
    <h2 class="blind">배너 광고 영역</h2>
    <div class="banner__inner">
        ....
    </div>
</section>

h2 태그 제목은 웹 표준을 준수하기 위해서 필요합니다. 하지만 우리 눈에는 필요가 없습니다. 즉 시각장애인을 위한 시각 리더기에서는 읽어야 하고, 우리 눈에는 안보이게 효과를 주어야 합니다. 그래서 블라인드 효과10를 줄 것입니다. 

.blind {
    position: absolute;
    clip: rect(0 0 0 0);
    width: 1px;
    height: 1px;
    margin: -1px;
    overflow: hidden
}

블라인드 효과는 다른 유형에서도 사용할 것이므로 나중에 공통요소로 뺄 것입니다. position 속성을 주어 위치를 설정하고, clip 속성을 이용하여, 영역을 0으로 설정합니다. 영역을 1px로 설정하고 안보이게 마진을 -1px로 설정하고, 지정한 영역이외에는 보이지 않도록 overflow: hidden를 설정하였습니다. 이렇게 하면 우리 눈에는 보이지 않는 효과가 생깁니다. 웹 표준을 준수하기 위해서 만들어낸 속성입니다. 

.section {
    padding: 120px 0;
}
.gmarket {
    font-family: "GmarketSans";
}

    font-weight: 300;
    text-decoration: underline;
    text-underline-position: under;
}

CSS 부분도 어떤게 좋다고 확실하게 말 할 수 없습니다. 지금은 구조가 간단하니 왼쪽이 좋아 볼 수도 있지만 구조가 복잡해지고, 여러명에 작업한다면, 오른쪽이 더 효율적일 수 있습니다. 코딩 할 때 제일 중요한 점은 가독성이 좋아야 하고, 유지 보수가 용이해야 합니다. 사이트를 만드는 것도 중요하지만, 나중에 수정할 것을 가정하고 코딩을 하는 것도 중요합니다.
